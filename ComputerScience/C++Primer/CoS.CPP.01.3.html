프로그램이 너무 복잡해지기 전에, C++가 주석(comments)을 다루는 방식을 살필 필요가 있다. 주석은 프로그램의 가독성을 증진하며, 알고리즘 요약, 변수 목적 정체화, 코드 파편의 명료화 등에 쓰인다. 컴파일러가 주석을 무시하기에, 주석은 프로그램의 행동 및 수행에 영향을 끼치지 않는다.

컴파일러가 주석을 무시하더라도, 사람은 그렇지 않다. 프로그래머는 시스템 문서가 낡았더라도 주석만큼은 믿는 경향을 보인다. 부정확한 주석은 주석이 아예 없는 것보다 나쁜데, 오도적일 수 있기 때문이다. 코드를 바꿀 때면, 주석 또한 바꿀 것!

C++ 주석 종류

C++에는 두 가지 종류의 주석이 있다. 단일 라인(single-line) 주석과 짝(paired) 주석이 그것이다. 단일 라인 주석은 두 개의 슬래쉬(//)로 시작해 개행으로 끝난다. 슬래쉬 오른쪽에 놓이는 것은 컴파일러에 의해 무시된다. 이와 같은 주석에는 추가적인 두 개의 슬래쉬를 포함한 모든 텍스트가 포함될 수 있다.

다른 종류의 주석은 두 개의 구분 문자(delimiters, /*, */)를 사용하는데, 이는 C에서 비롯한다. 이와 같은 주석은 /*로 시작해 */로 끝난다. 이들 주석은 */을 제외한, 대신 개행을 비롯한 모든 텍스트를 포함할 수 있다. 컴파일러는 /*와 */ 사이의 모든 것을 주석의 일부로 간주한다.

주석 쌍은 탭, 스페이스, 개행이 허용되는 곳이라면 어디든 허용된다. 주석 쌍은 프로그램에서 여러 라인을 차지할 수도 있지만 그러지 말 것이 권장된다. 주석 쌍이 여러 라인에 걸쳐 나타난다면, 안쪽 라인이 다중 라인 주석의 일부라는 점을 시각적으로 암시하는 편이 좋다. 우리의 스타일은 주석에서 각 라인을 애스터리스크(*)로 시작하여, 전체 범위가 다중 라인 주석의 일부라는 점을 암시하는 것이다.

프로그램은 주로 두 가지 주석 양식을 섞어 쓴다. 짝 주석은 다중 라인 확장에 주로 쓰이는데, 여기서는 두 개의 슬래쉬가 라인의 절반이나 단일 라인을 표시하기 위해 쓰이는 경향을 보인다. 

#include 
/*
 * Simple main function:
 * Read two numbers and write their sum
 */
int main ()
{
    // prompt user to enter two numbers
    std::cout << "Enter two numbers:" << std::endl;
    int v1 = 0, v2 = 0;   // variables to hold the input we read
    std::cin >> v1 >> v2; // read input
    std::cout << "The sum of " << v1 << " and " << v2
              << " is " << v1 + v2 << std::endl;
    return 0;
}

메모

이 책에서는 일반적인 프로그램 텍스트에서 주석이 두드러지게끔 주석을 초기화할 것이다. 실제 프로그램에서는 사용 프로그램 환경의 취향에 따라 주석 텍스트가 프로그램 코드 텍스트에서 두드러지는지 아닌지 결정된다.

묶이지 않는 주석 쌍

/*로 시작하는 주석은 */로 끝난다. 결과적으로, 하나의 주석 쌍은 다른 주석 쌍 속에 나타날 수 없다. 이런 실수로 인해 발생하는 컴파일러 오류 메시지는 의미심장하고 혼란스럽다. 가령, 다음 프로그램을 컴파일해볼 것. 
/*
 * comment pairs /* */ cannot nest.
 * "cannont nest" is considered source code,
 * as is the rest of the program
 */
int main ()
{
    return 0;
}
우리는 종종 디버깅 과정에서 코드 블럭을 주석으로 만들어야 할 수도 있을 것이다. 코드가 묶인 주석 쌍을 포함할 수 있기 때문에, 코드 블럭을 주석으로 만드는 가장 좋은 방법은 단일 라인 주석을 각 라인 시작 부분에 삽입하는 것이다. 
// /*
// *everything inside a single-line comment is ignored
// *including nested comment pairs
// */
