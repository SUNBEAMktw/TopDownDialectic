모든 C++ 프로그램은 하나 이상의 함수(functions)로 구성되며, 하나의 함수는 main이라고 명명되어야 한다. 운영체제는 main을 호출하여 C++ 프로그램을 실행한다. 아래는 main의 단순한 판본이다. 아무것도 안 하지만 운영체제에 값(value)을 반환(return)한다. 

int main()
    {
        return 0;
    }

함수 선언의 네 가지 요소로는 반환 타입(return type),함수 이름, 괄호로 묶인 (공백이 허용되는) 매개변수 목록(parameter list), 함수 몸체(function body)가 있다. 물론 main이 어떤 의미에서는 특별하지만, 우리는 main을 여느 함수와 마찬가지 방식으로 정의할 것이다.

이 예제에서 main은 (아무것도 없는 ()를 통해 확인할 수 있는) 빈 매개변수 목록을 갖는다. §6.2.5에서 main을 위해 정의할 수 있는 매개변수 타입을 논의할 것이다.

함수 정의의 마지막 단계인 함수 몸체란 문장(statements)의 한 블록(block)으로, 중괄호(curly brace)로 시작하고 끝난다. 


    {
        return 0;
    }

이 블록에서 유일한 문장은 return으로, 함수를 종결하는 문장이다. 이 경우처럼 return은 함수의 호출자에게 값을 돌려줄 수도 있다. return 문장이 값을 포함할 때, 반환 값은 함수의 반환 타입과 호환되어야 한다. 이 경우, main의 반환 타입은 int이며 반환 값은 0으로, int다.

메모
return 문장 끝에 있는 세미콜론(semicolon)에 주목할 것. 세미콜론은 C++의 문장 대부분에서 끝을 표시한다. 무시하기 쉽지만, 까먹으면 의미심장한 컴파일러 오류 메시지로 이어질 것이다.

대부분의 시스템에서 main에서 반환된 값은 상태 지시사(status indicator)다. 반환 값 0은 성공을 지시한다. 0이 아닌 반환 값은 시스템에 의해 정의된 뜻을 갖는다. 보통 0이 아닌 반환 값은 발생한 오류의 유형을 지시한다.

핵심 개념: 타입
타입은 프로그래밍의 가장 근본적인 개념 가운데 하나이며 이 책에서 계속 돌아올 개념이다. 타입은 자료의 내용과 그 자료상에서 가능한 작동을 정의한다.

우리의 프로그램이 조작하는 자료는 변수에 저장되며 모든 변수는 타입을 갖는다. v라는 이름의 변수 하나가 T라는 타입을 가질 때, 우리는 종종 “v는 타입 T를 갖는다”라고 말하거나 “v는 T다”라고 말한다.
