방금 우리는 1에서 10까지 수를 더하는 프로그램을 작성했다. 이 프로그램을 논리적으로 연장하면 사용자에게 덧셈을 수행할 숫자 집합을 입력하도록 요청하는 것일 테다. 이 경우, 우리는 얼마나 많은 수를 더해야 하는지 모른다. 대신, 더는 읽을 수가 없을 때까지 수를 읽을 것이다. 

#include <iostream>
int main()
{
    int sum - 0, value = 0;
    // read until end-of-file, calculating a running total of all values read
    while (std::cin >> value)
        sum += value; // equivalent to sum = sum + value
    std::cout << "Sum is: " << sum << std::endl;
    return 0;
}

여기 다음과 같은 입력을 준다면 

3 4 5 6

출력은 다음과 같을 것이다. 

Sum is: 18

main 내부의 첫 줄은 int 변수 두 개를 정의한다. 각각 이름은 sum과 value이며, 우리는 이를 0으로 초기화한다. 우리는 입력을 통해서 읽는 숫자 각각을 포착하기 위해 value를 사용할 것이다. 우리는 while 조건 내부에서 자료를 읽는다. 

while (std::cin >> value)

while 조건 평가 과정은 아래 표현식을 실행시킨다. 

std::cin >> value

이 표현식은 표준 입력에서 다음 수를 읽고 이를 value에 저장한다. 입력 연산자는 자신의 좌측 피연산자를 반환하는데, 이 경우 좌측 피연산자는 std::cin이다. 이 조건은, 따라서, std::cin을 검사한다.

istream을 조건으로서 사용할 때, 그 효과는 스트림의 상태를 검사하는 것이다. 스트림이 타당하다면, 즉 스트림이 오류와 조우하지 않았다면, 검사는 성공한 것이다. istream은 end-of-file에 도달하거나 정수가 아닌 값을 읽는 등 부당한 입력을 입력했을 때 부당하다. 부당한 상태에 놓인 istream은 조건이 거짓을 취하게 만들 것이다.

따라서, 우리의 while은 우리가 end-of-file에 (또는 입력 오류)에 조우할 때까지 실행될 것이다. while 몸체는 복합 대입 연산자를 사용하여 현재 값을 증진하는 sum에 더한다. 조건이 실패하면, while은 끝난다. 우리는 루프를 탈출하고 다음 문장이 실행된다. 즉 sum이 출력되고 endl이 이어지는 것이다.

키보드로 end-of-file 진입하기

키보드로 프로그램에 입력을 삽입할 때, 운영체제에 따라 end-of-file을 나타내는 관행이 다르다. 윈도우즈 시스템에서는 컨트롤 z를 누른 다음 엔터나 리턴 키를 누르면 된다. Mac OS X를 포함한 유닉스 시스템에서 end-of-file은 주로 컨트롤 d다.

컴파일 재고

컴파일러의 책무 가운데 하나는 프로그램 텍스트상에서 오류를 찾는 것이다. 컴파일러는 프로그램이 하는 일이나 저자의 의도를 파악할 수 없지만 프로그램의 형식(form)을 통해 오류를 감지할 수 있다. 아래는 컴파일러가 감지할 가장 일반적인 유형의 오류다.

구문 오류(Syntax errors): 프로그래머가 C++ 언어에서 문법적 오류를 저지른 경우. 아래 프로그램은 일반적인 구문 오류를 보여준다. 각 주석은 해당 라인의 오류를 설명한다. 

// error: missing ) in parameter list for main
int main( {
    // error: used colon, not a semicolon, after endl
    std::cout << "Read each file." << std::endl:
    // error: missing quotes around string literal
    std::cout << Update master. << std::endl;
    // error: second output operator is missing
    std::cout << "Write new master." std::endl;
    // error: missing ; on return statement
    return 0
}

타입 오류: C++에서 각각의 자료는 타입을 갖는다. 10이라는 값은 가령 int 타입을 갖는다. (달리 말해, "int다") "hello"라는 단어는 따옴표를 포함하여 스트링 리터럴이다. 일어날 수 있는 타입 오류의 예는 int 전달인자(argument)가 요구되는 함수에 스트링 리터럴을 넘기는 것이다.

선언 오류: C++에서 쓰이는 모든 이름은 사용 전에 반드시 선언되어야 한다. 이름 선언 실패는 주로 오류 메시지로 이어진다. 가장 흔한 선언 오류 예 두 가지로는 라이브러리에서 이름을 사용하는 과정에서 std:: 사용을 까먹는 것과 변수 이름 오타를 꼽을 수 있다. 

#include <iostream>
int main()
{
    int v1 = 0, v2 = 0;
    std::cin >> v >> v2;  //  error: uses "v" not "v1"
    //  error: cout not defined; should be std::cout
    cout << v1 + v2 << std::endl;
    return 0;
}

오류 메시지는 주로 라인 넘버와 컴파일러가 틀렸다고 보는 것에 관한 간략한 설명으로 구성된다. 보고된 순서에 따라 오류를 수정하는 것은 좋은 연습이다. 하나의 오류로 전부 무너지고 컴파일러가 실제로 일어난 오류보다 더 많은 오류를 보고하게 만들 수 있다. 수정할 때마다 코드를 재컴파일하는 것 또한 좋은 생각이다. 아니면 명백한 오류 조금만 수정한 다음 컴파일하거나. 이런 사이클은 편집-컴파일-디버그(edit-complie-debug)이라고 불린다.
